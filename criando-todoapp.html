Como construir um aplicativo Todo List com JavaScript puro

Construir um aplicativo de lista de tarefas é famoso exercício para aprender conceitos úteis que são amplamente aplicáveis ​​em muitos contextos de programação.

Pré-requisitos

Este tutorial presume que você tenha um conhecimento básico de JavaScript. Essencialmente, você precisa saber o que são variáveis, arrays, funções e objetos, mas não precisa ter experiência anterior com a construção de aplicativos JavaScript.

Iniciando

O aplicativo de lista de tarefas que construíremos neste tutorial será bem básico, porém, talvez o mais extenso aplicativo para quem está dandos os primeiros passos de programação.

Passo 1: Adicionar uma tarefa

Após a criação do projeto, vamos iniciar a edição do app.js, arquivo principal para esse tutorial. A primeira coisa que precisamos fazer é criar um array onde colocaremos os itens da lista de tarefas. Cada item de tarefas será um objeto com três propriedades: text:, com texto que usuário digitar, checked, um booleano para se uma tarefa foi marcada como concluída e id, um identificador.

Quando uma nova tarefa é adicionada, criamos um novo objeto (todo), inserimos no array (todoItems) e, adiante, exibiremos o valor da propriedade text na tela. Quando um todo for marcado como concluído, alternamos a propriedade checked para true, e quando o usuário exclui um todo, vamos localizar o item (todo) no array todoItems usando seu id removê-lo.

Vamos começar adicionando um item a fazer à nossa lista. Para fazer isso, precisamos ouvir o evento submit no elemento do formulário e, em seguida, invocar uma nova função addTodo() quando o formulário for enviado. 

// Este é o array que vai os itens a fazer (TODO)
let todoItems = [];

// Esta função vai criar um novo objeto tarefa(ToDo) baseado no
// texto que foi digitado no input text , e adiciona-lo dentro
// do array `todoItems` 
function addTodo(text) {
    const todo = {
        text,
        checked: false,
        id: Date.now(),
    };
  
  todoItems.push(todo);
  console.log(todoItems);
}

  // Seleciona o elemento form
const form = document.querySelector('.js-form');
// Add o submit event listener
form.addEventListener('submit', event => {
  // evitar submissão com refresh
  event.preventDefault();
  // seleciona o input text
  const input = document.querySelector('.js-todo-input');

  // Pega o valor do input e remove espaços em branco
  const text = input.value.trim();
  if (text !== '') {
    addTodo(text);
    input.value = '';
    input.focus();
  }
});

Por padrão, quando um formulário é enviado, o navegador tentará enviá-lo a um servidor, o que causará uma atualização da página. Para evitar que isso aconteça, podemos interromper o comportamento padrão ouvindo o evento submit no formulário e usando event.preventDefault().

Em seguida, selecionamos a entrada de texto e removemos os espaços em branco da string com trim(), em seguida, salvamos em uma nova variável chamada text. Se a variável text não for igual a uma string vazia, passamos o texto para a função addTodo() definida mais acima. 

const todo = {
  text,
  checked: false,
  id: Date.now(),
};

Dentro da função, criamos um novo objeto para a tarefa(todo) e adicionamos as propriedades que mencionei anteriormente. A propriedade text é definida como argumento da função, o checked inicializada para false e id é inicializada para com o número de milissegundos decorridos desde 1º de janeiro de 1970. Assim id será exclusivo para cada item de tarefas(todo). 

todoItems.push(todo);
console.log(todoItems);

Por fim, a tarefa(todo) é adicionada por push ao array todoItems, posteriormente, o array é exibido no console. Após o addTodo(text), no eventListener do formulário, o valor da entrada de texto é apagado, e recebe uma string vazia. Na última instrução, o foco é colocado novamente no input, afim de facilitar a adição de novos tarefas(todo).

Para testar, adicione tarefas e visualize o todoItems o inspecionar do seu navegador. Você verá que cada item de tarefas é representado por um objeto.

Renderizar os itens de tarefas

Depois que um novo item de tarefas(todo) é adicionado ao array todoItems, queremos que o aplicativo seja atualizado com o novo item renderizado na tela. Podemos fazer isso facilmente anexando um novo elemento li para cada item ao elemento(ul) do DOM que tem classe .js-todo-list. Para conseguir isso, adicione a nova função renderTodo() função acima da addTodo():

Veja uma ideia do código final neste link 
function renderTodo(todo) {
  // Primeiro selecionar o elemento com a classe  `js-todo-list`
  const list = document.querySelector('.js-todo-list');

  // Usando o operator ternário para checar se `todo.checked` é true
  // Se sim, atribuir 'done' to `isChecked`. Senão, atribur uma string vazia
  const isChecked = todo.checked ? 'done': '';
  // Criando um elemento `li` e atribuindo ele ao `node`
  const node = document.createElement("li");
  // Setando o atributo class
  node.setAttribute('class', `todo-item ${isChecked}`);
  // Setando o atributo data-key  com o id do todo
  node.setAttribute('data-key', todo.id);
  // Setando o conteúdo do elemento `li` criado acima
  node.innerHTML = `
    <input id="${todo.id}" type="checkbox"/>
    <label for="${todo.id}" class="tick js-tick"></label>
    <span>${todo.text}</span>
    <button class="delete-todo js-delete-todo">
    <svg><use href="#delete-icon"></use></svg>
    </button>
  `;

  // adicionando o elemento ao DOM como o último filho do
  // elemento referenciado por na variável `list`
  list.append(node);
}

A função renderTodo() tem um objeto todo como seu único parâmetro. Ele constrói um nó li no DOM usando o método document.createElement(). Na próxima linha, o atributo class é definido como todo-item ${isChecked}. O valor de isChecked será uma string vazia se a propriedade checked no objeto todo for false. Caso contrário, será 'done'. Você verá o efeito dessa classe 'done' na próxima seção.

Em seguida, o atributo data-key também é definido no elemento li. Ele é definido com a propriedade id do objeto todo e será usado para localizar um item específico no DOM posteriormente no tutorial. Em seguida, o conteúdo do elemento li é definido usando o método innerHTML e, finalmente, o elemento li é inserido como o último filho do elemento ul com a classe .js-todo-list.

Alterando a linha console.log(todoItems) no addTodo() para renderTodo(todo) conforme mostrado abaixo para que a função renderTodo() seja chamada sempre que um novo item de tarefas for adicionado no todoItems. 

function addTodo(text) {
  const todo = {
    text,
    checked: false,
    id: Date.now(),
  };

  todoItems.push(todo);
  renderTodo(todo);
}

Experimente adicionando alguns itens de tarefas. Todos eles devem ser renderizados na página.

Marque uma tarefa como concluída

Vamos adicionar a capacidade de marcar uma tarefa como concluída. Para fazer isso, precisamos ouvir o evento click na caixa de seleção e alternar a propriedade checked no item de tarefa correspondente.

Adicione o seguinte código na parte inferior do painel JavaScript para detectar o item de tarefas que está sendo marcado: 

// Selecione a lista complete
const list = document.querySelector('.js-todo-list');
// Adicione o click event listener para a list e seus filhos
list.addEventListener('click', event => {
  if (event.target.classList.contains('js-tick')) {
    const itemKey = event.target.parentElement.dataset.key;
    toggleDone(itemKey);
  }
});

Em vez de ouvir cliques em cada elemento checkbox de seleção, estamos ouvindo cliques em todo o contêiner da lista(ul). Quando ocorre um evento de clique na lista(ul), uma verificação é feita para garantir que o elemento que foi clicado seja uma caixa de seleção(checkbox). Nesse caso, o valor do data-key no elemento pai da caixa de seleção é extraído e passado para uma nova função toggleDone() (mostrada a seguir) que deve ser colocada abaixo da função addTodo(). 

function toggleDone(key) {
  // findIndex é método de array que returna a posição do elemento
  // em um array.
  const index = todoItems.findIndex(item => item.id === Number(key));
  // Localiza o item no array todoItems e configura o atributo checked
  // para o oposto. Isto significado, `true` deve se tornar `false` and vice
  // versa.
  todoItems[index].checked = !todoItems[index].checked;
  renderTodo(todoItems[index]);
}

Esta função recebe a chave do item da lista que foi marcado ou desmarcado e encontra o item correspondente na todoItems usando o método findIndex. Assim que tivermos o índice do item, precisamos localizá-lo no array todoItems usando a notação de colchetes. O valor da propriedade checked no item todo é então definido com o valor oposto. Por fim, a função renderTodo() é chamada com o objeto de todo passado. Se você executar o código agora e tentar marcar um item, ele duplicará o item de tarefas em vez de marcar o existente.

Para corrigir isso, precisamos primeiro verificar se o item de tarefas atual existe no DOM e substituí-lo pelo nó atualizado, se existir. Altere a função renderTodo() conforme mostrado abaixo: 

function renderTodo(todo) {
  const list = document.querySelector('.js-todo-list');
  // selecione o atual item  de todo no DOM
  const item = document.querySelector(`[data-key='${todo.id}']`);

  const isChecked = todo.checked ? 'done': '';
  const node = document.createElement("li");
  node.setAttribute('class', `todo-item ${isChecked}`);
  node.setAttribute('data-key', todo.id);
  node.innerHTML = `
    <input id="${todo.id}" type="checkbox"/>
    <label for="${todo.id}" class="tick js-tick"></label>
    <span>${todo.text}</span>
    <button class="delete-todo js-delete-todo">
    <svg><use href="#delete-icon"></use></svg>
    </button>
  `;

  // Se o item já existir no DOM
  if (item) {
    // substituir
    list.replaceChild(node, item);
  } else {
    // senão, adicionar ele ao fim da lista
    list.append(node);
  }
}

Primeiro, o item de tarefa atual é selecionado(todo). Se existir no dom , o elemento será retornado e posteriormente substituído. Se o item não existir (como é o caso de novos itens a fazer), ele será adicionado ao final da lista.

Excluir itens de tarefas

Semelhante à forma como implementamos o último recurso, ouviremos os cliques no elemento .js-delete-todo, em seguida, pegaremos a chave do pai (li) e a passará para uma nova função deleteTodo que removerá o objeto de tarefa(todo) correspondente no todoItem enviaremos o item de tarefas renderTodo() para ser removido do DOM.

Primeiro, vamos detectar quando o botão de exclusão é clicado: 
const list = document.querySelector('.js-todo-list');
    list.addEventListener('click', event => {
      if (event.target.classList.contains('js-tick')) {
        const itemKey = event.target.parentElement.dataset.key;
        toggleDone(itemKey);
      }
    
      // ADD ESSE BLOCO
      if (event.target.classList.contains('js-delete-todo')) {
        const itemKey = event.target.parentElement.dataset.key;
        deleteTodo(itemKey);
      }
    });

Em seguida, crie uma função deleteTodo() abaixo de toggleDone() conforme mostrado abaixo: 

function deleteTodo(key) {
  // pesquisa o correspondente objeto todo no array todoItems
  const index = todoItems.findIndex(item => item.id === Number(key));
  // Cria um novo objeto com propriedades do atual todo item
  // e a propriedade `deleted` que é definido como verdadeiro
  const todo = {
    deleted: true,
    ...todoItems[index]
  };
  // remova o item todo a partir do array filtrado e uma nova atribuição a todoItems
  todoItems = todoItems.filter(item => item.id !== Number(key));
  renderTodo(todo);
}

A função renderTodo() também necessita ser atualizado como a seguir: 
function renderTodo(todo) {
  const list = document.querySelector('.js-todo-list');
  const item = document.querySelector(`[data-key='${todo.id}']`);

  // ADICIONE ESSE BLOCO IF
  if (todo.deleted) {
    // remove esse item do DOM
    item.remove();
    return
  }

  const isChecked = todo.checked ? 'done': '';
  const node = document.createElement("li");
  node.setAttribute('class', `todo-item ${isChecked}`);
  node.setAttribute('data-key', todo.id);
  node.innerHTML = `
    <input id="${todo.id}" type="checkbox"/>
    <label for="${todo.id}" class="tick js-tick"></label>
    <span>${todo.text}</span>
    <button class="delete-todo js-delete-todo">
    <svg><use href="#delete-icon"></use></svg>
    </button>
  `;

  if (item) {
    list.replaceChild(node, item);
  } else {
    list.append(node);
  }
}

Adicionar um prompt de estado vazio

Um estado vazio ocorre quando não há dados para mostrar no aplicativo. Por exemplo, quando o usuário ainda não adicionou uma tarefa (primeiro uso) ou quando o usuário apagou a lista. É importante levar em conta esse estado ao projetar um aplicativo.

Quando não houver tarefas a serem exibidas, adicionaremos um prompt que incentiva o usuário a adicionar uma nova tarefa. Este recurso pode ser implementado apenas com html e css .

Tiraremos vantagem do seletor :empty css para exibir o prompt condicionalmente apenas se nenhum item existir na lista.

Adicione o seguinte código para o prompt de estado vazio no painel html, conforme mostrado abaixo: 
<main>
  <div class="container">
    <h1 class="app-title">todos</h1>
    <ul class="todo-list js-todo-list"></ul>
    <!-- ADICIONE ESSE BLOCO AQUI -->
    <div class="empty-state">
      <svg class="checklist-icon"><use href="#checklist-icon"></use></svg>
      <h2 class="empty-state__title">Add your first todo</h2>
      <p class="empty-state__description">What do you want to get done today?</p>
    </div>
    <!-- FIM DO BLOCO ADICIONADO -->
    <form class="todo-form js-form">
      <input autofocus type="text" aria-label="Enter a new todo item" placeholder="E.g. Build a web app" class="js-todo-input">
    </form>
  </div>
</main>




/* ADICIONE ABAIXO DOS OUTROS ESTILOS */
  
  .empty-state {
    flex-direction: column;
    align-items: center;
    justify-content: center;
    display: flex;
  }
  
  .checklist-icon {
    margin-bottom: 20px;
  }
  
  .empty-state__title, .empty-state__description {
    margin-bottom: 20px;
  }


Embora pareça normal, o problema é que a mensagem persiste mesmo quando uma tarefa é adicionada à lista. O comportamento pretendido é que a mensagem desapareça depois que uma tarefa for adicionada e só reapareça quando não haver mais tarefas a serem exibidas.

Este trecho de CSS nos dará o que queremos: 
/* Altere `display: flex` para `display: none` */
    .empty-state {
      flex-direction: column;
      align-items: center;
      justify-content: center;
      display: none;
    }
    
    /* Add this below the other styles */
    .todo-list:empty {
      display: none;
    }
    
    .todo-list:empty + .empty-state {
      display: flex;
    }

O elemento .empty-state está oculto por padrão com display: none e só aparece quando .todo-list está vazio. Estamos usando o seletor :empty para detectar quando .todo-list está vazio e o seletor irmão(sibling)(+) para direcionar .empty-state e aplicar display: flex a ele apenas quando .todo-list está vazio.

Persistir os dados do aplicativo

Nosso aplicativo de lista de tarefas está praticamente completo neste ponto, mas vamos adicionar mais um recurso para tornar as coisas um pouco mais interessantes e realistas. No momento, uma vez que a página é atualizada, todos os itens a fazer são apagados. Vamos evitar isso persistindo o estado do aplicativo no armazenamento local do navegador .
Adicione esta linha na parte superior da sua função renderTodo(): 

localStorage.setItem('todoItemsRef', JSON.stringify(todoItems));

Apenas strings podem ser armazenadas no localStorage, portanto, precisamos primeiro converter nossa todoItems em uma string JSON antes de passá-la para o método setItem que adiciona um novo item de dados na chave especificada.
Cada vez que a função renderTodo() é chamada, o valor de todoItemsRef no localStorage será substituído pelo valor atual da todoItems. Dessa forma, o array e a referência do localStorage correspondente são mantidas em sincronia.
Você pode testar isso abrindo as ferramentas de desenvolvimento do navegador, navegue até a guia Aplicativo e monitore a seção Armazenamento local. Se você não estiver usando o Chrome, as ferramentas de desenvolvimento podem ser organizadas de forma diferente.

A etapa final é renderizar todos os itens da lista de tarefas existentes quando a página for carregada. Adicione o seguinte snippet de código na parte inferior do painel JavaScript: 

document.addEventListener('DOMContentLoaded', () => {
    const ref = localStorage.getItem('todoItemsRef');
    if (ref) {
      todoItems = JSON.parse(ref);
      todoItems.forEach(t => {
        renderTodo(t);
      });
    }
  });

Quando o DOMContentLoadedevento é disparado, procedemos para recuperar o valor de todoItemsRefdo localStorage. Se existir, o JSON.parsemétodo é usado para converter a string JSON de volta à sua forma de array original e salvá-la em todoItems. Em seguida, renderTodo() é invocado para cada objeto de tarefa(todo) presente na array. Isso faz com que todos os itens de tarefas salvos sejam renderizados assim que a página carregar.
